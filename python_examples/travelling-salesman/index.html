Subject


The Travelling Salesman Problem describes a salesman who must travel between N cities. The order in which he does so is something he does not care about, as long as he visits each once during his trip, and finishes where he was at first.

Methods

In order to find the best solution, an evolutionary algorithm is used. It is based on a population of trajectory and the fitness function is the total distance. The main goal is to select the individual which has the smaller total distance. If an individual from the next generation has a better fitness function then he is selected (see the graph finess function according to each generation). 4 parameters can be defined: the number of generations, the mutation rate, the population size and the elite size (part of population which has the best fitness function).

The datasets have been taken from the site: university of Waterloo and then have been converted in csv files. You can find it in the same Github repository than the code used. The repository name is: travelling-salesman
